# YT Music Scrobbler

Dockerized service that periodically fetches your YouTube Music watch history and scrobbles it to Last.fm. Designed to capture plays from your phone (or any device) that your browser scrobbling extension misses.

## Features

- Polls YT Music history on a configurable interval
- Deduplicates against your Last.fm recent scrobbles (won't re-scrobble what your browser extension already sent)
- Tracks history position to only scrobble new plays
- Runs as a lightweight Docker container

## Setup

### 1. Install dependencies for setup

```bash
pip install ytmusicapi
```

### 2. Authenticate with YT Music

```bash
python setup.py --ytmusic
```

This uses `ytmusicapi`'s browser auth flow. You'll paste request headers from an authenticated YouTube Music browser session. The resulting `config/browser.json` is used to access your history.

See: https://ytmusicapi.readthedocs.io/en/stable/setup/browser.html

### 3. Configure Last.fm

```bash
python setup.py --lastfm
```

Or run `python setup.py` to do both steps at once.

You'll need a Last.fm API application — create one at https://www.last.fm/api/account/create.

### 4. Run

```bash
docker compose up -d
```

View logs:

```bash
docker compose logs -f
```

## Configuration

All config is via environment variables (set in `.env`):

| Variable | Description | Default |
|---|---|---|
| `LASTFM_API_KEY` | Last.fm API key | required |
| `LASTFM_API_SECRET` | Last.fm API secret | required |
| `LASTFM_SESSION_KEY` | Last.fm session key (generated by setup, no password needed) | required |
| `POLL_INTERVAL` | Seconds between history polls | `300` |
| `HISTORY_LIMIT` | Number of history items to fetch per poll | `50` |

## How deduplication works

1. **History position tracking**: On each poll, the service compares the current YT Music history against the last-seen top track. Only entries that appeared since the last poll are considered new.

2. **Last.fm cross-check**: Before scrobbling, each track is checked against your Last.fm recent scrobbles. If your browser extension already scrobbled it, the service skips it.

3. **First-run safety**: On the very first run, no tracks are scrobbled — the service records the current history position and starts tracking from there.

## File structure

```
├── config/
│   └── browser.json    # YT Music auth (generated by setup)
├── data/
│   └── scrobbled.db    # SQLite state database (auto-created)
├── .env                # Last.fm credentials (generated by setup)
├── docker-compose.yml
├── Dockerfile
├── scrobbler.py
└── setup.py
```
