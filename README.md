# YT Music Scrobbler

Dockerized service that periodically fetches your YouTube Music watch history and scrobbles it to Last.fm. Designed to capture plays from your phone (or any device) that your browser scrobbling extension misses.

## Features

- Polls YT Music history on a configurable interval
- Smart deduplication against your browser extension's scrobbles
- Fuzzy matching on artist names and titles (handles `feat.`, remixes, multi-artist strings)
- Passes album info so Last.fm can display album art
- Syncs liked status — tracks liked on YT Music get loved on Last.fm
- Runs as a lightweight Docker container

## Setup

### 1. Authenticate with YT Music

```bash
docker compose run --rm setup-ytmusic
```

This uses `ytmusicapi`'s browser auth flow. You'll paste request headers from an authenticated YouTube Music browser session. The resulting `config/browser.json` is used to access your history.

See: https://ytmusicapi.readthedocs.io/en/stable/setup/browser.html

### 2. Configure Last.fm

```bash
docker compose run --rm setup-lastfm
```

You'll need a Last.fm API application — create one at https://www.last.fm/api/account/create. The setup will give you a URL to authorize the app in your browser (no password needed).

### 3. Run

```bash
docker compose up -d
```

View logs:

```bash
docker compose logs -f
```

### Development

```bash
docker compose -f docker-compose.dev.yml up --watch
```

This enables debug logging and live-reloads `scrobbler.py` on save.

## Configuration

All config is via environment variables (set in `.env` or `config/.env`):

| Variable | Description | Default |
|---|---|---|
| `LASTFM_API_KEY` | Last.fm API key | required |
| `LASTFM_API_SECRET` | Last.fm API secret | required |
| `LASTFM_SESSION_KEY` | Last.fm session key (generated by setup, no password needed) | required |
| `POLL_INTERVAL` | Seconds between history polls | `300` |
| `HISTORY_LIMIT` | Number of history items to fetch per poll | `50` |
| `LASTFM_GRACE_PERIOD` | Seconds to wait after fetching YT history before checking Last.fm (gives the browser extension time to submit) | `5` |
| `LOG_LEVEL` | Log verbosity (`DEBUG`, `INFO`, `WARNING`, `ERROR`) | `INFO` |

## How deduplication works

### History tracking

The service stores an **anchor sequence** — the top 10 videoIds from the last poll. On each new poll, it finds where that sequence appears in the current history by matching 2+ consecutive IDs. Everything above the anchor is new. This handles song replays correctly: a single matching videoId could be a replay, but a consecutive sequence is the real boundary.

### Last.fm cross-check

Before scrobbling, new tracks are checked against recent Last.fm scrobbles to avoid duplicating what the browser extension already submitted. This uses:

- **Incremental fetching**: only scrobbles newer than the last-seen Last.fm timestamp are fetched, so we compare new-against-new
- **Grace period**: after fetching YT history, the service waits a few seconds before querying Last.fm, giving the browser extension time to submit
- **Fuzzy matching**: titles are compared with `(feat. ...)` and other parentheticals stripped; artists are split on `&`, `,`, `feat.` etc. so "TheFatRat & Cecilia Gault" matches "TheFatRat"

### First-run safety

On the very first run, no tracks are scrobbled — the service records the current history position and starts tracking from there.

## Deploying to another machine

Clone the repo and copy the `config/` directory (contains `browser.json` and `.env`). The `browser.json` cookies may expire over time — re-run `docker compose run --rm setup-ytmusic` if that happens. The Last.fm session key is permanent.

```
├── docker-compose.yml
├── docker-compose.dev.yml
├── Dockerfile
├── scrobbler.py
└── setup.py
```
